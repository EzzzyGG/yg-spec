# Research process

Папка `research/` — рабочий процесс поиска референсов/кандидатов (браузерные игры) для `yg4`.

## Файлы и назначение

- `00-sources.md` — источники (витрины/топы/аналитика) + «что снимать».
- `10-capture-template.md` — шаблон снятия витрины/топа (1 строка = 1 игра).
- `20-candidates-raw.md` — накопительная таблица всех кандидатов.
- `30-clusters.md` — кластеризация: `endless/campaign` + `core mechanic`.
- `40-shortlist-10.md` — shortlist 10 (5 endless + 5 campaign) + обязательные фичи (rewarded/interstitial/IAP/leaderboard/meta).
- `yg-research-merged.md` — текущий «сводный» документ (может быть временным/служебным).
- `yg-research-merged-snapshot.md` — снапшот для фиксации состояния на дату.

## Как обновлять (кратко)

1. **Снимаем витрины/топы** по `10-capture-template.md`.
2. Все находки складываем в `20-candidates-raw.md`.
3. Разносим кандидатов по кластерам в `30-clusters.md`.
4. Выбираем финальные 10 в `40-shortlist-10.md` и проверяем фичи монетизации/лидербордов/меты.
5. По необходимости обновляем `yg-research-merged.md` (как «человеческий» summary) и фиксируем снапшот.
