# Core loop / жанр / механики — требования (черновик)

Цель: зафиксировать **классификацию игры**, **жанр**, и **core loop**, чтобы дальше корректно спланировать Unity-реализацию, SDK/монетизацию и контент.

> Статус: Draft. Этот документ отвечает на пункт TODO: «Сделать классификацию по core loop / жанру / механикам».

## 1) Классификация (в терминах casual / hypercasual / hybrid)

### 1.1 Предлагаемая категория
- **Hybrid-casual endless** (endless-runner / lane-runner / obstacle-dodger) с метой.

Почему не чистый hypercasual:
- Планируется **meta progression** (апгрейды/перки), **IAP**, **лидерборды**, что обычно выводит проект за рамки «1-сессия и всё».

Почему не midcore:
- Сессии и управление остаются максимально простыми, быстрый старт, низкий порог входа.

### 1.2 Плей-сессия и цели
- Длительность одной попытки: **15–120 секунд** (с ростом при прогрессе).
- Микроцель: «прожить дольше / набрать больше очков».
- Метацель: «прокачать параметры, открыть улучшения, повышать рекорды».

## 2) Core loop (высокий уровень)

### 2.1 Основной цикл (1 попытка)
1. **Start run** (мгновенный старт)
2. **Avoid / collect** (уклонение от препятствий + сбор ресурсов/бонусов)
3. **Score grows** (очки/дистанция/мультипликатор)
4. **Fail** (столкновение/ошибка/таймер)
5. **Reward** (софт-валюта/прогресс/возможный rewarded video)
6. **Upgrade** (улучшения, перки, стартовые бонусы)
7. **Repeat**

### 2.2 Варианты «что игрок делает каждую секунду» (moment-to-moment)
Выбрать один и зафиксировать как «ядро»:
- **Lane switch** (3 полосы): свайп/кнопки влево-вправо.
- **Single-axis dodge** (влево-вправо по оси): удержание/перетаскивание.
- **Tap timing** (прыжок/рывок по таймингу): тапы по препятствиям.

Рекомендуется для WebGL/мобайл (простота ввода):
- Lane switch (3 lanes) или single-axis dodge.

## 3) Жанр и подписи (для документации)

Коротко:
- **Endless runner** / **Endless dodger**

Теги (под поиск/позиционирование):
- arcade, reflex, runner, endless, score attack

## 4) Механики (MVP)

### 4.1 Обязательные (must-have)
- Управление 1 жест/ось (keyboard + touch).
- Генерация препятствий/паттернов.
- Нарастание сложности (скорость/частота/комбинации).
- Сбор: 1–2 типа pickup (софт-валюта + временный бонус).
- Смерть/рестарт ≤ 2 секунды.

### 4.2 Бонусы (nice-to-have, но просто)
- Магнит на pickups (временный)
- Щит/вторая жизнь
- Мультипликатор очков по серии

### 4.3 Мета-прогрессия (связка с core loop)
- Софт-валюта за попытку.
- Перманентные апгрейды (скорость старта, шанс бонуса, длительность бонусов, стартовый щит).

## 5) Прогресс сложности

- Сложность растёт по времени/дистанции: каждые N секунд повышаем один из параметров:
  - скорость
  - плотность препятствий
  - сложность паттернов

## 6) Как это маппится на монетизацию (проверка консистентности)

- **Rewarded video**: после фейла — «воскреснуть» или «x2 награда».
- **Interstitial**: редкий показ между раннами (с частотным ограничением).
- **IAP**:
  - Remove ads
  - Small/Medium/Large RUB packs
  - Starter pack (выгодный набор)

## 7) Decision checklist (что нужно утвердить)

Отметь/выбери (оставь один вариант в каждом блоке):
- Core moment-to-moment: lane switch / axis dodge / tap timing
- Primary fail condition: collision / timer / both
- Primary score: distance / points / combo-based
- Picks: coin-only / coin+bonus
