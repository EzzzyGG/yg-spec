# YG SDK integration

## Init / bootstrap

- Интегрируем YG SDK на старте приложения.
- Приложение должно корректно стартовать в 2 режимах:
  - **Web + YG SDK доступен** (прод)
  - **Web без YG SDK** (локальная разработка / превью / блокировки)
- При отсутствии SDK используем **фоллбек-адаптер** (заглушки), чтобы core loop работал без рекламы/платежей/лидербордов.

### Источник данных / события

- Инициализация SDK и подписки на события должны быть изолированы в одном модуле (например, `YgSdkAdapter`).
- Должны быть события/колбэки:
  - `onReady` (SDK готов)
  - `onError` (ошибка инициализации)
  - `onAuthChanged` (пользователь залогинился/разлогинился)
  - `onLanguageChanged` (смена языка/локали)

## Env / Auth / Lang

### Env

- Определяем окружение:
  - `prod`: Yandex Games, SDK доступен.
  - `dev`: локальный запуск/стенды, SDK может отсутствовать.

### Auth

- Поддерживаем авторизацию игрока через YG (если доступно).
- Требования:
  - не блокировать игру, если пользователь не авторизуется;
  - хранить признак `isAuthorized` и `playerId` (если есть);
  - при смене статуса авторизации обновлять идентичность для аналитики и лидербордов.

### Lang

- Язык берём из SDK (если доступно), иначе используем дефолт `ru`.
- Точки применения:
  - тексты UI
  - цены/IAP описание

## Ads

### Rewarded video

- Используем rewarded ads для увеличения retention и monetization.
- Минимальные требования:
  - показы только по явному действию игрока;
  - таймаут/ошибка не ломают игровой цикл;
  - выдача награды **только после success callback**.

### Interstitial

- Интерстициалы показываем по правилам (пример):
  - не чаще X минут;
  - не в середине рана (только на экранах/переходах);
  - учитываем ограничения платформы.

## IAP

- Набор IAP (RUВ):
  - small / medium / large currency pack
  - remove ads
  - starter pack

### Требования

- Поток покупки:
  1) запрос продукта
  2) инициирование покупки
  3) подтверждение успеха
  4) выдача контента
  5) сохранение и аналитика

- Покупка должна быть идемпотентной (повторный колбэк не даёт двойную награду).

## Cloud saves

- Сохраняем игровой прогресс в облако (если доступно) и локально как фоллбек.

### Требования

- Версионирование сохранений (schema version).
- Конфликт-резолвинг (если есть локальное и облачное):
  - выбираем "самое новое" по timestamp или "самое далёкое" по прогрессу (определить политику).
- Автосейвы:
  - в конце рана
  - при покупке IAP
  - при важных апгрейдах

## Leaderboards

- Лидерборды завязаны на auth (если пользователь не авторизован — показываем гостевой режим).

### Требования

- Запись результата: по завершению рана/сессии.
- Чтение топа: при открытии экрана лидербордов.
- Фоллбек: если SDK недоступен — показываем локальный топ/плейсхолдер.

## Analytics

- Минимальная аналитика (через YG/внешний слой, если доступно):
  - `session_start`
  - `run_start` / `run_end`
  - `ad_rewarded_started` / `ad_rewarded_completed` / `ad_rewarded_failed`
  - `iap_started` / `iap_success` / `iap_failed`
  - `cloud_save_success` / `cloud_save_failed`
