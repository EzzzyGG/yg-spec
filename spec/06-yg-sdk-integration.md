## Интеграция YG SDK — чеклист (что дергаем и где)

Ниже — **плоский список интеграций**. Каждая интеграция описывает: **когда вызываем**, **что делаем**, **какие данные храним/передаём**, и **fallback**.

### 1) Env / Init

- Инициализация SDK при старте приложения
  - Когда: самый ранний момент после загрузки, до показа первого экрана.
  - Что: `sdk.init()` / `await sdk.ready()` (названия условные) + чтение окружения.
  - Данные: версия игры, build, платформа, locale.
  - Fallback: если SDK не доступен — работаем в "offline mode" (без сетевых фич), логируем.

### 2) Auth

- Авторизация пользователя
  - Когда: при входе в игру или при первом обращении к облаку/лидерам.
  - Что: `sdk.auth.getUser()` / `sdk.auth.signIn()`.
  - Данные: `userId`, `isGuest`, токен/сессия (если есть), признаки платформы.
  - Fallback: гостевой режим; все сетевые фичи — опциональны.

### 3) Lang / Locale

- Определение языка/локали
  - Когда: сразу после init.
  - Что: `sdk.getLocale()`.
  - Данные: `locale` → маппинг на наши локали (например `ru`, `en`).
  - Fallback: `en`.

### 4) Ads

- Rewarded video (за награду)
  - Когда: по кнопке "получить бонус" (например x2 награда / revive).
  - Что: `sdk.ads.showRewarded()`.
  - Данные: placement id (если есть), context (откуда вызвали).
  - Результат: success → выдаём награду; cancel/fail → без награды.

- Interstitial (межстраничная)
  - Когда: по триггеру частоты (например раз в N матчей/смертей/экранов).
  - Что: `sdk.ads.showInterstitial()`.
  - Данные: placement id, counters.
  - Ограничения: кап по времени/сессии.

### 5) IAP (покупки)

- Список товаров
  - Когда: при открытии магазина.
  - Что: `sdk.iap.getProducts()`.
  - Данные: product ids, цены/валюта.

- Покупка
  - Когда: по нажатию "купить".
  - Что: `sdk.iap.purchase(productId)`.
  - Данные: `productId`, `purchaseToken/receipt`.
  - Пост-условие: применяем entitlement (убрать рекламу/валюта/пак).
  - Fallback: ошибка/отмена — ничего не выдаём.

### 6) Cloud Saves (облачные сохранения)

- Загрузка сейва
  - Когда: после auth (или сразу после init, если auth implicit).
  - Что: `sdk.cloud.load(slot)`.
  - Данные: JSON состояния (версионируем: `saveVersion`).
  - Конфликт: выбираем по `updatedAt` или предлагаем пользователю (если требуется).

- Сохранение
  - Когда: по событиям прогресса (закончился забег, апгрейд, покупки) + периодически.
  - Что: `sdk.cloud.save(slot, data)`.
  - Данные: минимальный компактный JSON; без персональных данных.
  - Fallback: локальный сейв.

### 7) Leaderboards (таблицы лидеров)

- Отправка результата
  - Когда: конец run / матч.
  - Что: `sdk.leaderboards.submit(boardId, score, extra)`.
  - Данные: `boardId`, `score`, возможно `seasonId`.

- Получение топа / соседей
  - Когда: экран лидеров.
  - Что: `sdk.leaderboards.getTop(boardId, limit)` + `getAroundMe`.
  - Данные: `limit`, курсор/offset.
  - Fallback: показываем только локальные результаты.

### 8) Telemetry (если будет)

- События
  - Когда: ключевые точки (tutorial complete, purchase success, ad watched).
  - Что: `sdk.analytics.track(event, props)`.
  - Примечание: если аналитики в SDK нет — оставляем интерфейс для подключения later.
